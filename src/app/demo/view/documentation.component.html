Overview

PrimeNG Designer API is SASS based theme engine for creating PrimeNG themes easily.

Designer Application

This demo application is created to test your components while designing your theme. As it is an Angular CLI application so to run it locally, extract the contents of the designer zip file and run the following command.

ng serve

Then navigate to http://localhost:4200 to access the application in your local environment, now it is time to create your own theme.

Create Your Own Theme

A built-in base theme named "mytheme" is available to get started with your own theme using 500+ variables. To begin with navigate to the src/assets/mytheme/ folder that includes the following artifacts;

  • /fonts Folder of the font files
  • _fonts.scss: Definition of the @font-face of the theme
  • _variables.scss: List of variables to create your own theme
  • _extensions.scss: Customizations to the theme
  • theme.scss: Main theme file that imports the scss files required for the theme

You may now start designing your own theme by changing the variables, for example change the $primaryColor variable and save the _variables.scss file. Your new theme will be generated and browser will refresh the application automatically. Use the components menu at the sidebar to view your theme.

Customization

Designer API is located inside the src/assets/designer folder and an actual theme imports the main _core.scss from the designer folder. In case you need to make structural changes to the design system, add these changes to the _extensions.scss files instead of editing the .scss files inside the designer folder to avoid any maintenance issues. Designer is updated periodically for improvements and updating is done by replacing the designer folder only. For seamless updates, never edit the designer folder as src/assets/mytheme/_extensions.scss file is created for this purpose.

Font Family

Default font of the designer is "Open Sans" and it can be changed in three steps.

  • Placing the font artifacts such as eot, woff files inside the fonts.
  • Define the font-face at _fonts.scss.
  • Change the $fontFamily at _variables.scss.

Following example uses Roboto font for the theme.

_fonts.scss 
/* roboto-300 - latin */
@font-face {
    font-family: 'Roboto';
    font-style: normal;
    font-weight: 300;
    src: url('../fonts/roboto-v18-latin-300.eot'); /* IE9 Compat Modes */
    src: local('Roboto Light'), local('Roboto-Light'),
        url('../fonts/roboto-v18-latin-300.eot?#iefix') format('embedded-opentype'), /* IE6-IE8 */
        url('../fonts/roboto-v18-latin-300.woff2') format('woff2'), /* Super Modern Browsers */
        url('../fonts/roboto-v18-latin-300.woff') format('woff'), /* Modern Browsers */
        url('../fonts/roboto-v18-latin-300.ttf') format('truetype'), /* Safari, Android, iOS */
        url('../fonts/roboto-v18-latin-300.svg#Roboto') format('svg'); /* Legacy iOS */
}
/* roboto-regular - latin */

_variables.scss 
$fontFamily:"Roboto","Helvetica Neue",sans-serif;

Theme Size

Component sizes including the font-sizes, margins and paddings are defined using em units which allows changing the overall size, as a result changing the $fontSize variable is enough to scale the components. Default is 14px and example below creates a bigger theme based on a scale of 16px.

$fontSize:16px;

Build

Once the theme is ready for use, built the theme.css file and import it to your application. Once theme.css is generated inside mytheme folder, copy theme.css along with fonts folder to your application to use the theme. There are many ways to import theme.css in an Angular application, for example if you are using CLI, you may use the styles section at .angular-cli.json to define your css to be included in bundle. 

sass --update src/assets/mytheme
